# 014-建立平台类型体系

## 背景
当前Agent使用统一的写作提示词，无法针对不同平台（如知乎、公众号、小红书）和内容类型提供专业化写作能力。

## 目标
建立完整的平台类型体系，定义各平台的特征和写作要求。

## 验收标准
- [ ] 定义支持的平台列表（知乎、公众号、小红书、B站等）
- [ ] 明确各平台的文章特征（字数、行文风格、内容结构）
- [ ] 建立平台配置的数据结构
- [ ] 实现平台配置的加载和管理

## 实施记录

### 平台定义

#### 1. 微信公众号
- 字数：1500-5000字
- 风格：专业但易懂，观点鲜明
- 结构：开头吸引+中间3-5观点+结尾总结行动
- 特点：需要关注转化，语言简洁有力

#### 2. 知乎
- 字数：2000-8000字
- 风格：理性分析为主，兼顾可读性
- 结构：问题定义+原因分析+解决方案+案例支撑
- 特点：需要展现专业性，逻辑严密

#### 3. 小红书
- 字数：300-1000字
- 风格：轻松活泼，生活化
- 结构：痛点引入+解决方案+效果展示
- 特点：需要视觉化表达，真实感强

#### 4. B站
- 字数：1500-4000字
- 风格：幽默有梗，专业不失趣味
- 结构：开场吸引+干货输出+互动引导
- 特点：需要网感，弹幕友好
