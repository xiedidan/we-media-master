# 008-Trace/日志系统(UI进度+文件日志)

## 背景
当前Agent执行过程对用户不可见，问题难以定位。需要实现Trace系统，让用户看到执行进度。

## 目标
1. UI进度显示：搜索素材进度 → 生成文章进度
2. 文件日志：记录每次执行的输入输出
3. 方便调试和问题定位

## 验收标准
- [x] Streamlit界面显示执行进度步骤
- [x] 执行日志保存到logs/目录
- [x] 日志包含时间戳、输入、输出摘要
